Aula 2 – Algoritmos e Pseudocódigo

Algoritmos

Na aula anterior, entendemos que um algoritmo é uma sequência de passos bem definidos que levam a uma solução. Agora, vamos aprofundar esse conceito, aprendendo a representar algoritmos de duas formas muito utilizadas: pseudocódigo e fluxogramas.

Um algoritmo é composto basicamente por três partes:

  • Entrada – os dados ou informações que o algoritmo vai receber.
  • Processamento – as operações ou instruções que serão executadas.
  • Saída – o resultado obtido após o processamento.

Exemplo simples: calcular a média de duas notas.
Entrada: duas notas numéricas.
Processamento: somar as notas e dividir por dois.
Saída: mostrar a média final.

Pseudocódigo

Pseudocódigo é uma forma de escrever algoritmos em linguagem natural estruturada, ou seja, em português, mas de forma organizada e próxima da lógica das linguagens de programação. Ele não é executado pelo computador, mas serve como rascunho antes de escrever o código real.

Exemplo de pseudocódigo para trocar uma lâmpada:

inicio
    lampadaQueimada = sim
    lampadaNova = sim
    retirarLampadaQueimada() {
        pegarEscada = sim
        subirNaEscada = sim
        desrosquearLampadaQueimada = sim
        colocarLampadaNova()
    }
    colocarLampadaNova(){
        pegarLampadaNova = sim
        rosquearLampadaNova = sim
        descerDaEscada = sim
        testarLampada()
    }
    testarLampada(){
        pressioneOinterruptor = sim
        lampadaAcende = sim
    }
fim

Além do pseudocódigo, também é comum representar algoritmos através de fluxogramas, que utilizam símbolos gráficos para indicar o fluxo de execução. Essa forma visual ajuda muito a compreender a lógica antes de implementar o programa.

Nesta aula, aprendemos que algoritmos podem ser descritos de diferentes formas, seja em pseudocódigo ou em fluxogramas. Essas representações são fundamentais porque permitem planejar e visualizar a solução antes de escrever o código em PHP ou em qualquer outra linguagem.
